| `/works/OL262438W` | the-sign-of-the-four, the-sign-of-four — Sherlock Holmes true-duplicate |

**Root cause hypothesis:** the enrichment matcher (likely the OL search →
title-similarity step in `scripts/enrich-*-classification.py` or the
`merge-google-library.py` Google Books lookup feeding into Open Library
joins) accepts a fuzzy title hit without verifying author or distinguishing
multi-volume series. When the same query (e.g. "Capital") returns the same
Open Library work, every record with a similar title gets the same key.

**Suggested follow-up effort:**
1. Add a per-volume / per-author guard to the OL matching step.
2. Backfill: re-run `scripts/enrich-ol-firstedition.py` (or the relevant
   enricher) over the 119 affected records with the stricter matcher to
   reassign correct work keys.
3. Add a CI invariant in `test_data_integrity.py` that fails when an
   `ol_work_key` is shared by 2+ records whose titles' word-overlap ratio
   is below a threshold (e.g. 0.5).
